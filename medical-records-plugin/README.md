# Medical Records Management Plugin

## توضیحات افزونه
این افزونه برای مدیریت پرونده‌های پزشکی با یکپارچگی کامل با افزونه بوکلی طراحی شده است. پزشکان و بیماران از جداول بوکلی خوانده می‌شوند و امکان مدیریت کامل پرونده‌های پزشکی، ویزیت‌ها و سوابق پزشکی فراهم است.

## نصب و راه‌اندازی

1. پوشه `medical-records-plugin` را در مسیر `wp-content/plugins/` کپی کنید
2. از پیشخوان وردپرس، به بخش افزونه‌ها بروید
3. افزونه "Medical Records Management" را فعال کنید
4. پس از فعال‌سازی، جداول لازم به صورت خودکار ایجاد می‌شوند

## شورت‌کدها

### شورت‌کد پزشک
```
[medical_records_doctor]
```
این شورت‌کد را در صفحه‌ای قرار دهید که پزشکان باید به آن دسترسی داشته باشند. فقط کاربران با نقش پزشک یا مدیر می‌توانند این صفحه را مشاهده کنند.

### شورت‌کد بیمار
```
[medical_records_patient]
```
این شورت‌کد را در صفحه‌ای قرار دهید که بیماران باید به آن دسترسی داشته باشند. فقط کاربران وارد شده به سیستم می‌توانند این صفحه را مشاهده کنند.

## پنل مدیریت

در منوی پیشخوان، بخشی به نام "پرونده‌های پزشکی" اضافه می‌شود که شامل:
- لیست تمام بیماران از بوکلی
- امکان ایجاد پرونده برای بیماران
- مشاهده و حذف پرونده‌ها

## ویژگی‌ها

### برای مدیران و پزشکان:
- مشاهده لیست بیماران از بوکلی
- ایجاد پرونده پزشکی برای بیماران
- ثبت ویزیت جدید با اطلاعات کامل (علت مراجعه، تشخیص، داروها، فایل‌ها)
- فیلتر ویزیت‌ها بر اساس کلینیک، پزشک و تاریخ
- مشاهده جزئیات کامل ویزیت
- تقویم شمسی برای انتخاب تاریخ

### برای بیماران:
- مشاهده پرونده پزشکی شخصی
- مشاهده لیست ویزیت‌ها
- ثبت سوابق پزشکی (گروه خونی، بیماری‌های خاص، داروهای مصرفی)
- آپلود فایل‌های پزشکی

## ساختار پایگاه داده

### جدول mr_medical_records
- id: شناسه پرونده
- customer_id: شناسه بیمار از بوکلی
- doctor_id: شناسه پزشک از بوکلی
- blood_group: گروه خونی
- special_diseases: بیماری‌های خاص
- age: سن
- current_medications: داروهای مصرفی
- medical_files: فایل‌های پزشکی
- created_at: تاریخ ایجاد
- updated_at: تاریخ به‌روزرسانی

### جدول mr_visits
- id: شناسه ویزیت
- medical_record_id: شناسه پرونده پزشکی
- doctor_id: شناسه پزشک
- clinic_id: شناسه کلینیک
- visit_date: تاریخ ویزیت
- complaint: علت مراجعه
- diagnosis: تشخیص
- medications: داروهای تجویزی
- files: فایل‌های پیوست
- created_at: تاریخ ایجاد

## امنیت

- استفاده از nonce برای تمام درخواست‌های AJAX
- بررسی دسترسی کاربران برای هر عملیات
- sanitization و validation تمام ورودی‌ها
- لاگ خطاها برای دیباگ

## وابستگی‌ها

- افزونه Bookly باید نصب و فعال باشد
- jQuery
- Select2 (از CDN بارگذاری می‌شود)
- Persian Datepicker (از CDN بارگذاری می‌شود)

## توسعه

این افزونه به صورت ماژولار طراحی شده و به راحتی قابل گسترش است. فایل‌های اصلی:

- `medical-records-plugin.php`: فایل اصلی افزونه
- `includes/class-medical-records-db.php`: توابع پایگاه داده
- `includes/class-medical-records-ajax.php`: هندلرهای AJAX
- `includes/class-medical-records-shortcodes.php`: شورت‌کدها
- `includes/class-medical-records-admin.php`: توابع پنل مدیریت
- `assets/js/frontend.js`: جاوااسکریپت فرانت‌اند
- `assets/js/admin.js`: جاوااسکریپت پنل مدیریت
- `assets/css/frontend.css`: استایل‌های فرانت‌اند
- `assets/css/admin.css`: استایل‌های پنل مدیریت

## پشتیبانی

برای گزارش خطا یا درخواست ویژگی جدید، لطفاً با توسعه‌دهنده تماس بگیرید.
